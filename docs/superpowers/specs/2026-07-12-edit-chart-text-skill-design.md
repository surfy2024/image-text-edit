# 科研图表文字无痕替换 Skill 设计

## 1. 目标

创建一个完全本地离线运行的 Codex Skill。用户上传科研图表、地图或工程图后，只需用自然语言说明替换关系，例如“把图片中的 HZ 修改成 CS”，Skill 即可动态提取旧文字、新文字和修改范围，定位目标，修复旧文字背景，匹配原样式绘制新文字，并输出新图片。

`HZ → CS` 仅是交互示例。Skill 不预设任何固定文本。

## 2. 首版范围

### 包含

- PNG、JPG、JPEG 图片。
- 科研图表、地图和工程图中的英文、中文、数字及常见符号。
- 一次一组或多组替换。
- 唯一匹配、多个匹配和全部替换。
- 纯色、渐变色及局部线条背景。
- 原图保留、结果另存和 JSON 诊断报告。

### 不承诺

- 任意复杂照片背景上的文字替换。
- 恢复无法获得的原始商业字体。
- 可靠识别严重模糊或像素不足的文字。
- 修改文字之外的图形对象。

## 3. 用户交互

Skill 从每次用户指令动态生成替换清单：

```json
[
  {
    "old_text": "用户本次指定的旧文字",
    "new_text": "用户本次指定的新文字",
    "scope": "one | all | ask",
    "location_hint": null
  }
]
```

解析规则：

- 旧文字和新文字必须来自当前指令，不得写死示例值。
- 识别“全部”“所有”“左上角”“第二个”等范围或位置限定。
- 一句话可包含多组替换；按清单顺序执行，并在执行前检查修改区域冲突。
- 缺少旧文字或新文字时先询问，不猜测。
- 唯一高置信匹配可直接修改。
- 多处匹配且用户明确要求“全部”时直接修改全部。
- 多处匹配但范围未明确时，先提示匹配数量并询问。
- 低置信度或找不到精确匹配时，输出带编号的候选预览图供用户确认。

## 4. 技术路线

采用确定性的本地混合流水线：

- PaddleOCR：多尺度文字检测与识别。
- OpenCV：文字掩膜处理、背景分类和局部修复。
- Pillow：字体测量、颜色与位置匹配及文字重绘。

选择该路线的原因是科研图表需要保护曲线、点位、色标和边界等非文字像素。生成式整图编辑存在无意改变科研信息的风险，不作为首版默认方案。

参考：

- PaddleOCR：https://www.paddleocr.ai/latest/en/quick_start.html
- OpenCV inpaint：https://docs.opencv.org/4.x/d7/d8b/group__photo__inpaint.html
- Pillow ImageDraw：https://pillow.readthedocs.io/en/stable/reference/ImageDraw.html

## 5. 架构与职责

### 5.1 指令解析器

把自然语言转换为结构化替换清单，验证必要字段，并规范化范围和位置限定。

### 5.2 文字定位器

对原图和放大图执行多尺度 OCR。通过精确匹配、大小写受控匹配和受限模糊匹配生成候选框，并保存识别置信度。模糊匹配只能用于提出候选，不能在未确认时直接修改。

### 5.3 歧义决策器

根据匹配数量、用户范围和置信度决定直接执行、询问用户或生成编号预览图。

### 5.4 样式估计器

分析目标文字及邻近文字，估计字体类别、字号、粗细、颜色、透明度、旋转角度、字符间距和对齐方式。无法取得原字体时，从本地字体库选取像素形态最接近的候选。

### 5.5 像素编辑器

从目标框生成精细字形掩膜，避免把整个矩形当作擦除区。根据背景类型选择：

- 纯色：邻域稳健颜色采样。
- 平滑渐变：二维颜色插值。
- 局部纹理或线条：OpenCV 局部修复，并限制掩膜膨胀范围。

清除旧文字后，以估计样式绘制新文字。新文字更长时，依次尝试轻微调整字距、在允许阈值内缩小字号、扩展文字框。若会覆盖邻近图形对象，则停止并请求用户选择，不强行压缩。

### 5.6 质量检查器

- 再次 OCR，验证新文字可识别且旧文字已消失。
- 比较修改前后图片，确认允许区域之外的像素完全一致。
- 检查修改区域边缘是否出现明显矩形、色差或断线。
- 任一硬性检查失败时，任务状态为失败，不把半成品作为成功结果。

## 6. 执行接口

提供一个确定性命令脚本，概念接口如下：

```text
edit-chart-text <image-path> --replace <old-text> <new-text> --scope one|all
```

多组替换通过 JSON 请求文件传递，避免复杂命令行转义。Codex Skill 负责自然语言理解、歧义交互、脚本调用和结果展示；脚本负责 OCR、像素编辑、验证及报告生成。

## 7. 输出契约

- 永不覆盖原图。
- 成功图片默认保存为 `<原文件名>_edited.png`。
- 同目录生成 `<原文件名>_edit-report.json`。
- 报告包含输入摘要、候选位置、已执行替换、OCR 置信度、样式参数、背景修复方法、像素差异范围和质量检查结果。
- 歧义或低置信度时生成 `<原文件名>_candidates.png`，用编号标记候选位置。

## 8. 异常处理

- 未找到旧文字：不修改，列出相似识别结果。
- 多处匹配且范围不明确：不修改，询问用户。
- 新文字放不下：提供缩小字号、扩展文字框或取消修改的选择。
- 掩膜涉及图形对象：缩小掩膜重试；仍无法隔离则停止。
- OCR、字体或背景置信度过低：生成候选或诊断预览并等待确认。
- 运行错误或质量检查失败：保留原图和报告，不输出成功状态。

## 9. 测试计划

测试样例覆盖：

- 英文：`HZ → CS`。
- 编号：`P10 → P40`。
- 小数：`25.00 → 26.50`。
- 中文：`水温 → 底层水温`。
- 单个匹配、多个匹配、明确全部和范围不明确。
- 纯色、渐变色和线条穿过文字的背景。
- 新文字比旧文字更短、相同和略长。
- PNG、JPG、JPEG。
- 找不到目标、OCR 误识别、目标重叠和空间不足。

单元测试分别覆盖指令规范化、候选选择、掩膜生成、背景修复、文字布局和报告输出。集成测试使用固定图片与替换请求，验证输出文字、像素保护边界和失败状态。示例上传图作为首个真实验收样本。

## 10. 验收标准

- 正确提取用户每次指定的旧文字、新文字及修改范围。
- 唯一高置信匹配可自动完成；歧义场景不擅自修改。
- 目标文字正确替换，原图不被覆盖。
- 允许修改区域之外的像素保持不变。
- 正常查看比例下，修改区域无明显矩形补丁、突兀色差或断线。
- 成功和失败均有结构化报告；失败提示包含可执行的下一步。

## 11. Skill 目录建议

Skill 使用简短、动作导向的名称 `edit-chart-text`，计划包含：

```text
edit-chart-text/
├── SKILL.md
├── agents/
│   └── openai.yaml
├── scripts/
│   ├── edit_chart_text.py
│   ├── detect_text.py
│   ├── render_text.py
│   └── validate_edit.py
├── references/
│   └── troubleshooting.md
└── assets/
    └── fonts/
```

仅在确有需要时放入可再分发字体；默认优先扫描本机字体，避免复制受许可限制的字体文件。
